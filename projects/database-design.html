ETL Data Pipeline

An end-to-end ETL pipeline that extracts data from public APIs, transforms and cleans it, then loads it into a PostgreSQL database with automated scheduling using Apache Airflow.

Timeline 2024
Role Solo Developer
Type Personal Project
Status Completed

Overview

This project demonstrates a complete ETL (Extract, Transform, Load) workflow built from scratch. The pipeline connects to a public REST API, processes the raw JSON data, applies transformation logic, and stores clean records in a PostgreSQL database.

The entire process is orchestrated using Apache Airflow, which schedules and monitors each step of the pipeline automatically.

Key Features

  • Automated data extraction from REST APIs using Python requests
  • Data cleaning and transformation with Pandas
  • Structured loading into PostgreSQL with error handling
  • Airflow DAGs for scheduling and monitoring
  • Logging and basic data quality checks

What I Learned

  • How to design and build a real ETL pipeline from scratch
  • Working with APIs and handling JSON data structures
  • Database schema design and SQL query optimization
  • Using Apache Airflow for workflow orchestration
  • Importance of error handling and data validation
