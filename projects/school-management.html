RDMS Course Project · Royal University of Phnom Penh
Click to enlarge

The New School Information System (SIS) is a complete database-driven system designed from scratch to replace an outdated manual school management process. Built as a group academic project for the RDMS course at RUPP, the system centralizes all school operations — from student enrollment and grade management to attendance tracking, fee payments, and automated notifications.

Project Overview

The school currently faces critical operational problems: no centralized student data, a fully manual enrollment process, poor communication between teachers and parents, and no integration between academic and financial systems. This system solves all of these by providing a fully normalized relational database with 15 tables serving 8 stakeholder groups.

Database Normalization

  • 1NF — All columns contain atomic values, no repeating groups anywhere
  • 2NF — All non-key attributes depend fully on the entire primary key
  • 3NF — No transitive dependencies: each attribute depends only on the primary key
  • Composite primary key used in STUDENT_ACTIVITIES (student_id + activity_id)
  • All foreign key relationships enforced at database level

Sample SQL Queries

-- Get all enrolled students with their class and course info
SELECT s.first_name, s.last_name,
       c.grade_level, co.course_name, e.enrollment_status
FROM STUDENTS s
JOIN ENROLLMENTS e ON s.student_id = e.student_id
JOIN CLASSES c    ON e.class_id = c.class_id
JOIN COURSES co   ON c.course_id = co.course_id
WHERE e.enrollment_status = 'Approved';
-- Get average grade per student per course
SELECT s.first_name, s.last_name,
       co.course_name, AVG(g.score) AS avg_score
FROM STUDENTS s
JOIN ENROLLMENTS e  ON s.student_id = e.student_id
JOIN CLASSES c      ON e.class_id = c.class_id
JOIN COURSES co     ON c.course_id = co.course_id
JOIN GRADES g       ON e.enrollment_id = g.enrollment_id
GROUP BY s.student_id, co.course_id
ORDER BY avg_score DESC;
-- Check unpaid fees per student
SELECT s.first_name, s.last_name,
       f.fee_type, f.amount, f.due_date, f.status
FROM STUDENTS s
JOIN FINANCIAL_FEES f ON s.student_id = f.student_id
WHERE f.status = 'Unpaid'
ORDER BY f.due_date;

Security Design

  • Role-Based Access Control (RBAC) — 4 roles: Admin, Teacher, Student, Parent
  • Password Hashing — All passwords stored using bcrypt (never plaintext)
  • HTTPS Encryption — All data in transit encrypted
  • Sensitive Data Encryption — Medical records encrypted at rest
  • Audit Logging — All important actions tracked by admin
  • Daily Backup — Automated backups logged in BACKUP_RECORDS table

What I Learned

This project was my deepest hands-on experience with relational database design. My specific role was designing the Entity Relationship Diagram (ERD) and reviewing the system processes — which required understanding every table relationship, cardinality, and data flow across all 15 tables. I learned how to translate business requirements into a normalized database schema, how to use composite primary keys correctly, how to enforce referential integrity through foreign keys, and how to design a system that serves multiple stakeholder groups with different access levels. Working in a team also taught me how to coordinate database design decisions across different parts of a system.
