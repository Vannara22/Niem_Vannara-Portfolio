ETL — Customer Call List
Data Cleaning Pipeline

Personal Data Engineering Project

This project demonstrates a complete ETL (Extract, Transform, Load) data cleaning pipeline applied to a raw customer call list dataset. The goal is to transform messy, inconsistent raw data into a clean, structured dataset ready for CRM use or downstream analysis — solving the common real-world problem of dirty data.

Project Overview

Real-world datasets are rarely clean. This project tackles the most common data quality issues found in customer datasets: inconsistent formatting, missing values, duplicate records, invalid characters, mixed data in single columns, and business logic filtering. Each issue was identified, analyzed, and resolved systematically using Python and Pandas.

Data Cleaning Steps

Step Action Details
01Load & InspectRead CSV, check head/tail, shape, dtypes, describe()
02Standardize ColumnsLowercase column names, replace spaces with underscores
03Remove DuplicatesDrop all duplicate rows across the dataset
04Missing ValuesCount and identify null values per column
05Clean Last NamesStrip special characters: periods, slashes, underscores, digits
06Format Phone NumbersRemove non-numeric chars, reformat to XXX-XXX-XXXX
07Split AddressSplit combined address into street, state, zip_code columns
08Standardize Valuespaying_customer & do_not_contact: Yes→Y, No→N
09Business FilterRemove all rows where do_not_contact = Y
10Drop Invalid PhonesRemove rows with missing or blank phone numbers
11Reset IndexRe-index after filtering to maintain clean row numbering
12Export Clean DataSave final cleaned dataset to customer_call_list_cleaned.csv

Key Challenges Solved

  • Phone numbers had mixed formats — some with dashes, symbols, or letters — all standardized to XXX-XXX-XXXX
  • Last names contained trailing special characters like ..., /, _ from data entry errors
  • Address was stored as a single string — split into 3 separate usable columns
  • Do-not-contact list filtering applied as a business rule to protect customer preferences
  • Mixed representations of Yes/No values standardized across multiple columns

Results & Output

  • ✅ All duplicate records removed
  • ✅ Phone numbers consistently formatted as XXX-XXX-XXXX
  • ✅ Address field split into street, state, and zip_code
  • ✅ Opted-out customers (do_not_contact = Y) fully filtered
  • ✅ Rows with invalid/missing phone numbers removed
  • ✅ Clean production-ready CSV exported successfully

What I Learned

This project taught me how to design a systematic data cleaning pipeline that mirrors real ETL workflows used in industry. I learned how to use Pandas string methods and regex for messy text data, apply business logic rules as data filters, restructure columns for better usability, and export clean data for downstream processing. Most importantly, I learned that data cleaning is not just technical — it requires understanding the business context behind each data field.
