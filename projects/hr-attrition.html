HR Employee Attrition
Analysis & Prediction

Personal Machine Learning Project

This project analyzes HR employee attrition data to identify key factors that drive employees to leave a company, and builds 8 machine learning classification models to predict whether an employee is likely to stay or leave. Employee attrition is a critical and costly business problem — this project uses data science to help HR teams understand and proactively address turnover risks.

Project Overview

Using a real HR dataset with 35 employee features (age, income, overtime, job satisfaction, distance from home, etc.), this project walks through a complete ML pipeline: data exploration, cleaning, visualization, model training, evaluation, and hyperparameter tuning. The focus is on identifying which factors most strongly predict employee departure.

Exploratory Data Analysis (EDA)

Before building models, a thorough EDA was conducted to understand the dataset and identify which features are most associated with attrition.

  • Plotted histograms for all 30 numeric features (6×5 grid layout)
  • Attrition Distribution — revealed class imbalance: more employees stayed than left
  • Correlation Heatmap — identified strongly correlated numeric features
  • Attrition vs Monthly Income — lower income employees leave more often
  • Attrition vs Age — younger employees (25–35) show higher attrition
  • Attrition vs OverTime — employees working overtime leave significantly more
  • Attrition vs Distance from Home — longer commutes correlate with higher attrition
  • Satisfaction Scores — analyzed Job, Environment, Relationship, and Involvement satisfaction

Data Preprocessing

  • Dropped uninformative columns: EmployeeCount, EmployeeNumber, Over18, StandardHours
  • Encoded all categorical variables using Label Encoding
  • Applied Z-score StandardScaler for feature normalization
  • Stratified 70/30 train-test split to maintain class balance

Machine Learning Models

8 classification algorithms were trained and evaluated to find the best performer:

🔵 Gaussian Naive Bayes

Probabilistic baseline using feature independence assumption

🔵 Bernoulli Naive Bayes

Binary feature variant of Naive Bayes

🟢 Logistic Regression

Linear classification model as a strong baseline

🟡 K-Nearest Neighbors

Distance-based classifier with k=5 neighbors

🟠 Decision Tree

Rule-based interpretable tree model

🔴 Random Forest

200-tree ensemble — strong generalization

🟣 SVC (RBF Kernel)

Support Vector Classifier with radial basis function

🟣 Linear SVC

Linear boundary SVM for high-dimensional data

Model Evaluation

Each model was evaluated using 5 metrics. Recall was prioritized as the most important metric — missing an employee who will leave (false negative) is more costly to a business than a false alarm (false positive).

  • Accuracy — overall percentage of correct predictions
  • Precision — how many predicted leavers actually left
  • Recall — how many actual leavers were correctly identified ⭐ key metric
  • Specificity — how well the model identifies employees who stay
  • F1 Score — harmonic mean of precision and recall
  • Confusion Matrix — detailed breakdown per model

Hyperparameter Tuning

Applied GridSearchCV with 5-fold cross-validation to fine-tune the SVC model. Tuned parameters: C (regularization), kernel (linear, rbf, poly), and gamma (scale, auto). The grid search identified the optimal combination of parameters that maximized cross-validated performance.

Key Findings

💰 Income is a major driver

Employees with lower monthly income show significantly higher attrition rates. Compensation is one of the strongest predictors of turnover.

⏰ Overtime doubles attrition risk

Employees who frequently work overtime are far more likely to leave — suggesting burnout as a key factor.

👶 Young employees leave more

Employees aged 25–35 show the highest attrition rates — likely seeking better opportunities or growth.

😔 Low satisfaction = higher risk

Low job satisfaction, environment satisfaction, and low involvement all strongly correlate with attrition.
